# Owlzops Mapper — JSON Field Reference

This document describes the schema of the JSON output generated by `owlzops-mapper audit --format json`.
This reference is intended for engineers integrating the mapper into CI/CD pipelines, SIEMs, or custom dashboards.

> **Note:** Fields may be omitted or return `null` if the scanner lacks sufficient privileges (e.g., run without `sudo`) or if the specific subsystem is not present on the host.

---

## Top-Level Object

| Field | Type | Description |
|---|---|---|
| `version` | String | The version of owlzops-mapper that generated the report. |
| `timestamp` | String (ISO 8601) | Exact time the scan was completed (UTC). |
| `hostname` | String | The system hostname (`/etc/hostname`). |
| `provider` | String | Detected cloud provider (e.g., `AWS`, `Hetzner`, `DigitalOcean`, `Unknown`). |
| `risk_score` | Integer (0-100) | Calculated risk score based on critical security findings (lower is better). |
| `exit_code_simulated` | Integer | What the CLI exit code would be (`0` = Clean, `1` = Critical issues, `2` = Non-root). |

---

## 1. System (`system`)

| Field | Type | Description |
|---|---|---|
| `os_release` | String | The human-readable OS name (e.g., `Ubuntu 22.04.4 LTS`). |
| `kernel_version` | String | The active Linux kernel version. |
| `uptime_seconds` | Integer | System uptime in seconds. |
| `cpu_cores` | Integer | Total logical CPU cores available. |
| `ram_total_mb` | Float | Total system memory in megabytes. |
| `load_average` | Array `[1m, 5m, 15m]` | System load averages. |
| `lsm_active` | Array of Strings | Active Linux Security Modules (e.g., `["AppArmor", "Yama"]`). |

---

## 2. Security (`security`)

| Field | Type | Description |
|---|---|---|
| `ssh_root_login` | String | Effective SSH root login status (`yes`, `no`, `prohibit-password`). |
| `ssh_password_auth` | Boolean | True if SSH allows password authentication. |
| `firewall_active` | Boolean | True if ufw, firewalld, iptables, or nftables has active block/drop rules. |
| `sudo_nopasswd_entries` | Array of Strings | Users/Groups found in `/etc/sudoers` allowing `NOPASSWD`. |
| `sysctl_issues` | Array of Strings | Detected insecure sysctl parameters (e.g., IP forwarding enabled, syncookies disabled). |
| `oom_kills_detected` | Integer | Count of Out-Of-Memory (OOM) process kills found in kernel logs. |
| `auditd_active` | Boolean | True if `auditd` is installed and running. |

---

## 3. Network (`network`)

| Field | Type | Description |
|---|---|---|
| `listening_ports` | Array of Objects | List of open ports. See **Port Object** below. |
| `ssl_certificates` | Array of Objects | Discovered local SSL certs and their expiry dates. |

### Port Object
* `port`: (Integer) The listening port number.
* `bind_address`: (String) e.g., `127.0.0.1` (safe) or `0.0.0.0` / `::` (exposed).
* `process_name`: (String) The daemon holding the port (e.g., `nginx`).

---

## 4. Docker (`docker`)

| Field | Type | Description |
|---|---|---|
| `installed` | Boolean | True if Docker daemon is detected. |
| `containers_running` | Integer | Count of currently running containers. |
| `dangling_images_mb` | Float | Disk space wasted by untagged/dangling images. |
| `containers` | Array of Objects | Detailed list of running containers. See **Container Object**. |

### Container Object
* `name`: (String) Container name.
* `image`: (String) Image used.
* `uptime`: (String) Human-readable uptime.
* `is_privileged`: (Boolean) True if running with `--privileged`.
* `bind_mounts`: (Array of Strings) Host-to-container volume mappings.

---

## 5. Storage (`storage`)

| Field | Type | Description |
|---|---|---|
| `mounts` | Array of Objects | Data for all physical filesystems (ignores loop/tmpfs). |

### Mount Object
* `mount_point`: (String) e.g., `/`, `/var/lib/docker`.
* `usage_percent`: (Integer) Disk space used (0-100).
* `inode_percent`: (Integer) Inodes used (0-100).

---

## 6. Databases (`databases`)

| Field | Type | Description |
|---|---|---|
| `detected_engines` | Array of Strings | E.g., `["PostgreSQL", "Redis"]`. Includes both native and Dockerized instances. |
| `data_dir_sizes` | Object | KV pairs of detected data directories and their size in MB. |

---

## 7. Backups & NTP

| Field | Type | Description |
|---|---|---|
| `backup_tools_detected` | Array of Strings | E.g., `["restic", "borg", "crontab:rsync"]`. |
| `ntp_synchronized` | Boolean | True if system clock is actively synchronized via systemd-timesyncd, chrony, or ntpd. |
